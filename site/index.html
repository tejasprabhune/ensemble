ensemble

A framework for building RL environments where several simulated users and several agents share one typed world. You write the world and the scenarios in Python; the framework handles message routing, tool dispatch, the trace log, and the grader that turns each rollout into a reward.

Why it exists

Most agent benchmarks model one agent talking to one user in a stateless sandbox. Production support, sales, customer success, and operations all look very different. There are multiple users contending for attention, multiple agents seeing each other's tool calls, shared state that changes underfoot, and policies that only become visible across many turns. Ensemble lets you build that, run rollouts at scale, and turn the rollouts into training signal.

What the on-ramp looks like

From a fresh clone, three commands take you from nothing to a working trace. The scaffold defaults to pure Python; the Rust path is the upgrade for worlds that need typed state with snapshot and restore semantics. 

uv sync
cargo build -p ensemble-cli
./target/debug/ensemble init my_world
cd my_world
./target/debug/ensemble run my_world.smoke

The Quickstart walks the rest of the way: pointing at a real model, looking at the trace, writing your first tool, comparing many configurations with the sweep runner, and the cross-run observability subcommands. The guides on the left cover each surface in depth.
