Observability

Every ensemble run lands in its own directory and appends one row to a flat traces/runs.jsonl index. Once that is true, cross-run questions become cheap: list recent runs, dump one run's meta, diff two runs' scores side by side, open two traces scroll-synced by tick, or export the index to JSON or CSV for downstream analysis. This guide walks through the on-disk layout and the subcommands that read it.

The on-disk layout

A run produces a per-run directory under traces/ and appends one row to the shared index. A flat symlink at the old path keeps the README quickstart and any tooling that hard-codes traces/<scenario>.jsonl working. 

traces/
  runs.jsonl                                                # append-only index, one row per run
  20260520T143022_my_world_smoke_a1b2c3d4/                  # per-run directory
    trace.jsonl                                             # the events the viewer reads
    meta.json                                               # the row above, duplicated for offline use
  my_world_smoke.jsonl -> 20260520T143022_my_world_smoke_a1b2c3d4/trace.jsonl

The run id is a sortable timestamp plus an 8-hex disambiguator. ls traces/ shows the newest run last, and tab completion still distinguishes runs of the same scenario at the same second.

A row in runs.jsonl

The index is one JSON object per line. Each row carries the fields you need to filter, sort, and compare without opening the per-run meta.json: 

{
  "run_id": "20260520T143022_my_world_smoke_a1b2c3d4",
  "scenario": "my_world.smoke",
  "world": "my_world",
  "backend": "mock",
  "started_at": 1779326173.89,
  "finished_at": 1779326175.90,
  "duration_s": 2.01,
  "scores": {"ok": 1.0},
  "costs": {},
  "trace_path": "traces/20260520T143022_my_world_smoke_a1b2c3d4/trace.jsonl"
}

The per-run meta.json carries the same fields, so a single run is self-describing even if you copy its directory out of the project.

Listing runs

ensemble runs list prints recent runs as a table sorted oldest-first. --scenario filters by scenario name; --limit caps the number of rows. 

ensemble runs list                                  # everything
ensemble runs list --scenario my_world.smoke        # one scenario
ensemble runs list --scenario my_world.smoke --limit 5
run_id                                                   scenario          when                 scores
-----------------------------------------------------------------------------------------------------------
20260520T143022_my_world_smoke_a1b2c3d4                  my_world.smoke    2026-05-20 14:30:22  ok=1.0
20260520T144210_my_world_smoke_b2c3d4e5                  my_world.smoke    2026-05-20 14:42:10  ok=0.5

The global --traces-dir flag points at a directory other than ./traces. Pass a sweep's directory to list its cells the same way.

Showing one run

ensemble runs show prints one run's meta as pretty-printed JSON. The argument is the full run id or any unique prefix. If the prefix matches more than one run, the command reports the matches and exits non-zero so you can retry with a longer prefix. 

ensemble runs show 20260520T143022           # full id
ensemble runs show a1b2c3d4                  # unique suffix works too
ensemble runs show 2026                      # ambiguous, lists matches and exits non-zero

Comparing two runs

ensemble runs compare diffs two runs' scores side by side. Each metric appears once with the A value, the B value, and the signed delta. The bottom section repeats the cost dicts verbatim so you can spot token or USD swings. 

ensemble runs compare a1b2c3d4 b2c3d4e5
A: 20260520T143022_my_world_smoke_a1b2c3d4  scenario=my_world.smoke  when=2026-05-20 14:30:22
B: 20260520T144210_my_world_smoke_b2c3d4e5  scenario=my_world.smoke  when=2026-05-20 14:42:10

metric                                       A               B  delta
--------------------------------------------------------------------------------
ok                                          1.0             0.5      -0.500
speed                                       0.8             0.9      +0.100

costs:
  A: {"tokens_in": 4500, "usd": 0.013}
  B: {"tokens_in": 4800, "usd": 0.014}

Pair the numeric diff with ensemble trace compare below to see the actual messages that produced the score delta.

Exporting the index

ensemble runs export emits the whole index in one shot. --format json is the default and writes a JSON array of meta records. --format csv flattens scores into one column per metric so the result loads into pandas without re-parsing JSON. 

ensemble runs export --format json > runs.json
ensemble runs export --format csv  > runs.csv
run_id,scenario,world,backend,finished_at,duration_s,score.ok,score.speed
20260520T143022_my_world_smoke_a1b2c3d4,my_world.smoke,my_world,mock,1779326175.9,2.01,1.0,0.8
20260520T144210_my_world_smoke_b2c3d4e5,my_world.smoke,my_world,mock,1779327730.1,2.05,0.5,0.9

The compare viewer

ensemble trace compare opens a two-column browser view of two traces. Each column renders one trace's message events (agent and user messages, tool calls, tool results, system notes) as a chronological feed. The sync-scroll toggle, on by default, ties the columns by tick so equivalent moments in both runs sit at the same vertical position. 

ensemble trace compare \
  traces/20260520T143022_my_world_smoke_a1b2c3d4/trace.jsonl \
  traces/20260520T144210_my_world_smoke_b2c3d4e5/trace.jsonl \
  --port 8765

The compare assets ship embedded in the binary so the command works offline without --site. --site <dir> is still honoured for live editing of the HTML and JS during development.

Filtering and ad-hoc analysis

The index is flat JSONL on purpose: a researcher can read it directly in any language without going through the CLI. The common pattern is to load the rows, filter to the scenario you care about, and aggregate over the scores dict: 

import json
from pathlib import Path

rows = [
    json.loads(line)
    for line in Path("traces/runs.jsonl").read_text().splitlines()
    if line.strip()
]

recent = [r for r in rows if r["scenario"] == "my_world.refund"][-20:]
mean_refund = sum(r["scores"]["refund_issued"] for r in recent) / len(recent)
print(f"refund_issued over last 20 runs: {mean_refund:.2f}")

Sweep results live in the same shape: each cell is a row in the sweep's index.jsonl, so the same snippet works against a sweep directory with one path change.
